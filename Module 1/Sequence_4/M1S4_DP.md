# **Déroulé pédagogique – M1 S4**

## Propulser son Book avec GitHub Pages

---

## **1 Consolidation Git & GitHub via GitHub Skills – 45 min**

**Objectifs :**

- Renforcer les manipulations Git essentielles (branches, commits, PR, merge).

- Travailler en autonomie grâce au module interactif GitHub Skills.

- Réinvestir les acquis de M1S3 pour gagner en fluidité.

*(Supports : GitHub – Portfolio & progression + GitHub Skills)*  

**Activités :**

- Démarrage guidé du module **“Introduction to GitHub”** :
  
  - suivre les instructions du bot GitHub,
  
  - créer une branche,
  
  - modifier un fichier,
  
  - ouvrir une Pull Request,
  
  - répondre aux commentaires,
  
  - réaliser le merge final.

---

## **2 Créer et publier son portfolio avec GitHub Pages – 1 h 15**

Basé sur : *GitHub – Portfolio*, *Ressources Webdesign*, *Ressources JS*

**Objectifs :**

- Publier un site personnel en ligne via GitHub Pages.

- Comprendre le fonctionnement d’un site statique (HTML / Markdown / CSS).

- Structurer la première version d’un Book professionnel.

**Activités :**

- Lancement du module **“GitHub Pages”** sur GitHub Skills.

- Publication automatique du site à l’adresse :
  
  ```text
  https://nom-utilisateur.github.io
  ```

- Ouverture du dépôt associé → repérage des fichiers clés (`index.md` ou `index.html`).

- Ajout d’un premier contenu structuré :
  
  - présentation,
  
  - compétences actuelles,
  
  - projets en cours,
  
  - objectifs.

- Exploration guidée des ressources :
  
  - inspirations design (Dribbble, Behance, Minimal Gallery) →
  
  - palettes & typographies (Coolors, Google Fonts) →
  
  - idées d’effets JS simples (Typed.js, Hover.css) →

**Livrables :**

- Première version du portfolio en ligne, accessible publiquement.

- Un contenu minimal mais structuré (4 sections).

---

## **3 Personnaliser son Book : esthétique & identité – 1 h 30**

Basé sur : *Ressources Webdesign* et *Ressources Créatives JS*

**Objectifs :**

- Apprendre à donner une identité visuelle simple mais professionnelle.

- Savoir intégrer CSS, images, typos et effets très légers.

- Découvrir comment rendre son Book évolutif au fil de la formation.

**Contenu abordé :**

- Organisation d’un petit site statique : `index.html`, `/assets/css`, `/assets/img`.

- Choix d’une palette cohérente + sélection de 2 typographies (ex : titres / texte).

- Ajout d’illustrations ou avatars libres (si souhaité).

- Intégration d’un petit effet JS optionnel :
  
  - typing text (Typed.js)
  
  - reveal au scroll (ScrollReveal / AOS)
  
  - hover animations pour les boutons

**Activités :**

- Atelier “amélioration du Book” :
  
  - mise en place d’un style minimaliste,
  
  - intégration Google Fonts,
  
  - choix d’une section mise en valeur (projets ou objectifs),
  
  - ajout d’un mini-effet JS discret.

- Utilisation guidée de plateformes d’inspiration :
  
  - OnePageLove, Lapa Ninja, Minimal Gallery →

**Livrables :**

- Portfolio niveau 1.5 :
  
  - style cohérent,
  
  - typographies installées,
  
  - au moins 1 amélioration visuelle significative,
  
  - site publiquement accessible et mis à jour.

---

## **4 Découverte de Codewars & entraînement algorithmique – 1 h 30**

Basé sur : *GitHub – Portfolio & progression* →

**Objectifs :**

- Introduire Codewars comme routine d’entraînement autonome et régulier.

- Comprendre l’intérêt d’entraîner la logique (exos courts, répétitifs, progressifs).

- Apprendre à lire une énoncé court et proposer une solution simple en JS.

**Travail à réaliser :**

- Création d’un compte Codewars.

- Paramétrage en JavaScript.

- Découverte des katas **8 kyu** (niveau débutant).

- Réalisation en direct de 2–3 katas simples, guidés par le formateur.

- Explication des étapes d’une résolution efficace :
  
  - lire l’exemple,
  
  - identifier l’entrée / sortie,
  
  - trouver la transformation simple,
  
  - tester.

**Livrables :**

- 3 katas résolus sur Codewars (minimum).

- Capture d’écran ou lien du profil.

---

# **Résumé clair du déroulé**

1. **GitHub Skills** (45 min)  
   PR, review, merge en autonomie pour consolider Git.

2. **Publication GitHub Pages** (1 h 15)  
   création + mise en ligne du portfolio développeur.

3. **Personnalisation du Book** (1 h 30)  
   design, typographies, couleurs, effets JS optionnels.

4. **Codewars** (1 h 30)  
   découverte, premiers katas, mise en place d’une routine d’entraînement.
