LES FORMULAIRES EN SYMFONY

Créer, traiter et valider des formulaires proprement

1. Pourquoi Symfony propose un système de formulaires ?

Dans une application web, les formulaires sont omniprésents :

  • création d’un compte
  • ajout / modification de données
  • recherche
  • authentification
  • administration

Symfony propose un système de formulaires structuré pour :

  • éviter le traitement manuel de $_POST
  • centraliser la validation
  • lier les formulaires aux entités
  • garantir une architecture propre

Le formulaire est un objet, pas un simple HTML.

2. Principe général d’un formulaire Symfony

Un formulaire Symfony repose sur trois éléments :

  1. une Entity (optionnelle mais fréquente)
  2. une classe FormType
  3. un contrôleur qui traite le formulaire

Schéma mental :

Formulaire HTML
   ↓
Request
   ↓
FormType
   ↓
Validation
   ↓
Entity / Service

3. La classe FormType

La logique d’un formulaire est définie dans une classe dédiée.

Par convention :

src/Form/

3.1 Exemple simple de FormType

namespace App\Form;

use App\Entity\Article;
use Symfony\Component\Form\AbstractType;
use Symfony\Component\Form\FormBuilderInterface;
use Symfony\Component\Form\Extension\Core\Type\TextType;
use Symfony\Component\OptionsResolver\OptionsResolver;

class ArticleType extends AbstractType
{
    public function buildForm(FormBuilderInterface $builder, array $options): void
    {
        $builder
            ->add('title', TextType::class)
        ;
    }

    public function configureOptions(OptionsResolver $resolver): void
    {
        $resolver->setDefaults([
            'data_class' => Article::class,
        ]);
    }
}

Cette classe :

  • décrit les champs
  • ne gère ni l’affichage, ni la Request
  • peut être réutilisée

4. Créer un formulaire dans un contrôleur

Dans le contrôleur, vous créez le formulaire à partir du FormType. 

$form = $this->createForm(ArticleType::class, $article);

Symfony :

  • instancie le formulaire
  • le lie à l’entité
  • prépare la gestion des données

5. Traiter la Request

Un formulaire Symfony doit être lié à la Request. 

$form->handleRequest($request);

Cette méthode :

  • détecte si le formulaire a été soumis
  • hydrate l’objet
  • déclenche la validation

5.1 Vérifier la soumission et la validité

if ($form->isSubmitted() && $form->isValid()) {
    // traitement
}

C’est le cœur du traitement des formulaires Symfony.

6. Exemple complet dans un contrôleur

#[Route('/article/new', name: 'article_new')]
public function new(
    Request $request,
    EntityManagerInterface $entityManager
): Response {
    $article = new Article();

    $form = $this->createForm(ArticleType::class, $article);
    $form->handleRequest($request);

    if ($form->isSubmitted() && $form->isValid()) {
        $entityManager->persist($article);
        $entityManager->flush();

        return $this->redirectToRoute('article_index');
    }

    return $this->render('article/new.html.twig', [
        'form' => $form,
    ]);
}

Le contrôleur :

  • orchestre
  • délègue au formulaire
  • délègue à Doctrine
  • reste lisible

7. Afficher un formulaire avec Twig

Twig fournit des fonctions dédiées.

{{ form_start(form) }}
{{ form_widget(form) }}
{{ form_end(form) }}

Symfony :

  • génère le HTML
  • gère les noms des champs
  • protège contre les attaques CSRF

8. Validation des données

La validation se fait au niveau de l’Entity, pas dans le contrôleur.

8.1 Exemple de contraintes

use Symfony\Component\Validator\Constraints as Assert;

#[ORM\Column(length: 255)]
#[Assert\NotBlank]
#[Assert\Length(min: 3)]
private string $title;

Symfony :

  • valide automatiquement lors de handleRequest
  • remonte les erreurs dans le formulaire

9. Afficher les erreurs dans Twig

Twig affiche automatiquement les erreurs :

{{ form_errors(form.title) }}

Ou globalement :

{{ form_errors(form) }}

10. Bonnes pratiques essentielles

  • un formulaire = une classe FormType
  • aucune validation dans le contrôleur
  • aucune logique métier dans Twig
  • redirection après succès (POST → Redirect → GET)
  • le formulaire manipule des objets, pas $_POST

11. Ce que vous devez maîtriser à ce stade

Vous devez être capable de :

  • expliquer le rôle d’un FormType
  • créer un formulaire simple
  • traiter la Request
  • valider les données
  • persister avec Doctrine
  • afficher un formulaire avec Twig

12. Ce que vous n’êtes pas encore censé maîtriser

  • formulaires imbriqués
  • collections dynamiques
  • formulaires complexes JavaScript
  • custom form themes

Ces points viendront plus tard.

13. À retenir absolument

Un formulaire Symfony est un objet. La Request est traitée par le formulaire. La validation est déclarative. Le contrôleur orchestre, le formulaire travaille.

Conclusion

Le système de formulaires est un pilier de Symfony.

Bien utilisé, il permet :

  • un code propre
  • une validation centralisée
  • une sécurité intégrée
  • une vraie séparation des responsabilités

La maîtrise des formulaires marque une étape décisive vers des applications Symfony professionnelles.
