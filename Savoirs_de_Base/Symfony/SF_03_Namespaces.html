LES NAMESPACES EN PHP

Comprendre et utiliser correctement namespace et use

1. Pourquoi les namespaces existent

En PHP, les namespaces servent à organiser le code et à éviter les conflits de noms entre les classes.

Sans namespace, deux classes portant le même nom ne peuvent pas coexister : 

class User {}
      class User {} // Erreur

Avec des namespaces :

namespace App\Entity;

      class User {}
      
namespace App\Security;

      class User {}

Les deux classes sont distinctes et peuvent être utilisées dans le même projet.

2. Un namespace n’est pas un chemin de fichier

Même si un namespace ressemble souvent à une arborescence de dossiers, il s’agit d’une structure logique, pas d’un chemin disque.

Exemple :

App\Controller\HomeController

Ce nom est une hiérarchie logique. Le lien avec les dossiers (src/Controller/HomeController.php) est assuré par Composer (autoload PSR-4), pas par PHP lui-même.

3. Pourquoi utilise-t-on le caractère \ ?

En PHP, le caractère \ est le séparateur de namespace.

Il est utilisé parce que :

  • / est réservé aux chemins système
  • . est l’opérateur de concaténation
  • \ permet une hiérarchie claire sans ambiguïté

Exemple :

Symfony\Component\HttpFoundation\Response

4. Déclarer un namespace

Un namespace se déclare au tout début du fichier PHP : 

<?php

      namespace App\Controller;

Toutes les classes définies dans ce fichier appartiennent à ce namespace.

5. Exemple concret : une classe avec namespace

namespace App\Controller;

      use Symfony\Bundle\FrameworkBundle\Controller\AbstractController;
      use Symfony\Component\HttpFoundation\Response;

      class HomeController extends AbstractController
      {
          public function index(): Response
          {
              return new Response('Bienvenue');
          }
      }

Le nom complet de cette classe est :

App\Controller\HomeController

6. Utiliser une classe définie dans un autre namespace

PHP ne “voit” pas automatiquement les classes des autres namespaces.

Si vous êtes dans un autre fichier et que vous écrivez :

new HomeController();

PHP cherchera une classe HomeController dans le namespace courant. S’il ne la trouve pas, une erreur est déclenchée.

6.1 La bonne pratique : utiliser use

Pour utiliser HomeController ailleurs dans le projet, vous devez importer sa classe : 

use App\Controller\HomeController;

Vous pouvez ensuite l’utiliser simplement :

$controller = new HomeController();

Le mot-clé use indique à PHP :

“Quand j’écris HomeController, je parle de App\Controller\HomeController.”

6.2 Utiliser le nom complet (sans use)

Il est aussi possible d’écrire directement :

$controller = new \App\Controller\HomeController();

Cela fonctionne, mais :

  • c’est plus long
  • moins lisible
  • déconseillé si la classe est utilisée plusieurs fois

6.3 Cas où use n’est pas nécessaire

Si le fichier courant est déjà dans le même namespace :

namespace App\Controller;

Alors vous pouvez écrire directement :

new HomeController();

PHP comprend que vous faites référence à :

App\Controller\HomeController

7. Le mot-clé use ne charge pas une classe

Point fondamental :

  • use n’inclut pas de fichier
  • use ne fait pas de require
  • use crée uniquement un alias de nom

Le chargement réel du fichier est assuré par l’autoloader de Composer.

8. Le namespace global

Certaines classes PHP natives se trouvent dans le namespace global.

Exemple :

new \DateTime();

Le \ au début indique que vous utilisez la classe native PHP, et non une classe portant le même nom dans votre projet.

9. Bonnes pratiques à respecter

  • Une classe par fichier
  • Toujours utiliser des namespaces dans un projet moderne
  • Importer les classes avec use
  • Éviter les classes sans namespace
  • Garder des namespaces cohérents avec l’architecture du projet

10. À retenir absolument

Un namespace organise les noms des classes. Il évite les conflits. Le caractère \ est un séparateur logique. Le mot-clé use permet d’importer une classe d’un autre namespace.

11. Résumé express

  • namespace : définit où se trouve votre classe
  • use : permet d’utiliser une classe d’un autre namespace
  • \ : sépare les niveaux du namespace
  • Les dossiers sont liés aux namespaces via l’autoload, pas par PHP

Conclusion

Les namespaces sont indispensables en PHP moderne et en Symfony. Les comprendre vous permet :

  • de lire le code plus facilement
  • de structurer proprement vos projets
  • d’éviter des erreurs difficiles à diagnostiquer
