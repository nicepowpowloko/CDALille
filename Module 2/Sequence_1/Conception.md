# Design et Conception d’Interfaces Utilisateur

## 1. Pourquoi concevoir avant de coder ?

* Le design et la conception permettent d’anticiper *l’expérience utilisateur (UX)* et *l’interface graphique (UI)* avant d’écrire une seule ligne de code ;
* Cela réduit les erreurs, les retours et les coûts de modification ;
* Pour le projet « mini-site réservation de chalet », cela signifie : définir la navigation, les types de pages, les besoins utilisateurs, avant de coder en Bootstrap ou en React.

---

## 2. Les grandes étapes de la conception

1. **Recherche / cadrage** : recueil des besoins, personas, parcours utilisateur.
2. **Architecture de l’information (IA)** : sitemaps, arborescence, logique de navigation.
3. **Wireframing (maquettes filaires)** : versions simples, faibles détails, seul le squelette.
4. **Maquette UI / prototypage** : couleurs, typographies, images, interactions.
5. **Tests & itérations** : feedback utilisateurs, ajustements.
6. **Handoff aux développeurs** : livrer les assets, les spécifications, l’export.

### Attention point important

L’**Architecture de l’information** (**IA**) dans le design est la discipline qui s’occupe de :

* organiser les contenus d’un site,
* structurer la navigation,
* définir les catégories,
* créer l’arborescence,
* clarifier les parcours utilisateur,
* rendre l’information facile à trouver et à comprendre.

**L’IA détermine comment les utilisateurs trouvent et accèdent à l’information et ne doit pas ici être confondue avec les LLMs.**
