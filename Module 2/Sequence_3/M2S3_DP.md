# Déroulé pédagogique – Contrôler la qualité de l’interface (vitesse & accessibilité)

**Objectif global :** comprendre les bases de la performance et de l’accessibilité, puis appliquer ces notions dans la continuité du travail HTML/CSS/Bootstrap.

---

## 1 Cours : Performance & Accessibilité

**Durée :** ~1h30
**Support :** DevTools + Lighthouse

**Contenu :**

* Identifier les éléments qui ralentissent une page (images, scripts, CSS inutile)
* Utiliser les outils navigateur : Network, Performance, Lighthouse
* Structuration sémantique pour l’accessibilité (titres, navigation, alt, contraste)
* Navigation clavier + tests rapides d’accessibilité

**Exercice rapide :**

* Repérer 2 problèmes de vitesse + 2 problèmes d’accessibilité sur une page simple et proposer une correction.

---

## 2 Continuation HTML / CSS / Bootstrap + TP Chalet Montagne

**Durée :** reste de la journée
**Support :** Projet “Chalet Montagne”

**Contenu :**

* Amélioration du site : structure, contrastes, alt, lisibilité
* Optimisation : poids des images, nettoyage CSS, bon usage des utilitaires Bootstrap
* Vérification du responsive et des comportements sur mobiles
* Application des retours Lighthouse (performance + accessibilité)

**Objectif :** rendre le projet plus propre, plus rapide et plus accessible.

---

## Résumé

1. **Cours performance & accessibilité**
2. **Application directe sur le projet Chalet Montagne**
3. **Améliorations HTML / CSS / Bootstrap pour un site plus rapide et plus accessible**
